# Бот заказов «по принципу таблицы остатков»

Прототип Telegram-бота: сотрудник выбирает точку → выбирает категорию (или «все
категории») → бот присылает **весь список позиций одним сообщением** (названия
и мин/макс уже проставлены) → сотрудник одним ответом вписывает все числа по
порядку (каждое на новой строке или через пробел) → бот сам считает заказ и
статус (🟢/🔴/🔵) — так же, как в вашей Excel-таблице (`Заказ = Макс − Факт`)
→ показывает сводку с возможностью заполнить заново → отправляет готовый заказ
с шаблоном конкретной точки.

## Файлы

| Файл | Назначение |
|---|---|
| `seed_products.xlsx` | **Единственный файл, который редактируете руками.** Лист «Позиции» — точки/категории/наименования/мин/макс. Лист «Точки» — куда слать готовый заказ и заголовок шаблона. |
| `import_products.py` | Загружает `seed_products.xlsx` в базу `order_bot.db`. Запускать заново после каждого изменения Excel. |
| `db.py` | Работа с SQLite (точки, каталог, история введённых остатков). |
| `bot.py` | Сам бот (aiogram 3). |

## Запуск

```bash
pip install -r requirements.txt

# 1. Заполните seed_products.xlsx под свои точки/позиции
# 2. Загрузите каталог в базу:
python import_products.py seed_products.xlsx

# 3. Получите токен у @BotFather в Telegram и запустите:
export BOT_TOKEN="ваш_токен_от_BotFather"
export ADMIN_IDS="ваш_telegram_id"   # без этого /admin доступен всем — см. раздел ниже
python bot.py
```

Бот будет отвечать в личных сообщениях и в группах, куда его добавят.
Заказы — командой `/start`, управление каталогом (категории/позиции/мин-макс) — командой `/admin`.

## Управление каталогом: два способа

**1. Через Excel** (`seed_products.xlsx` → `import_products.py`) — удобно для массового
первоначального заполнения или переноса большого списка позиций.

**2. Через саму Telegram-команду `/admin`** — теперь можно на ходу, без Excel:
- создавать и удалять категории
- создавать и удалять позиции внутри категории
- менять мин./макс. остаток у любой позиции
- переименовывать категории и позиции

⚠️ Учтите: если после правок через `/admin` вы запустите `import_products.py`
заново, он **перезапишет** каталог точки содержимым Excel-файла — изменения,
сделанные через бота, будут потеряны для этой точки. Используйте один способ
как основной, второй — для точечных правок между обновлениями Excel.

### Кто может пользоваться `/admin`

Доступ ограничивается переменной окружения `ADMIN_IDS` (Telegram user id через
запятую, узнать свой id можно у **@userinfobot**):

```bash
export ADMIN_IDS="123456789,987654321"
```

Если `ADMIN_IDS` не задан — команда `/admin` доступна вообще всем, кто пишет
боту (удобно для тестов, но небезопасно в проде — обязательно задайте
переменную перед реальным запуском).

### Сценарий работы с `/admin`

1. `/admin` → выбрать точку (или создать новую)
2. В меню:
   - **➕ Категория** — ввести название, категория сразу появится в каталоге (даже пустая — позиции можно добавить позже)
   - **➕ Позиция** — выбрать категорию (или создать новую тут же) → название → единица измерения → мин и макс через пробел (например `2 10`)
   - **✏️ Мин/Макс позиции** — выбрать категорию → позицию → ввести новые мин/макс
   - **✏️ Переименовать позицию / категорию**
   - **🗑 Удалить позицию / категорию** — с подтверждением (удаление категории удаляет и все её позиции)
   - **📋 Показать каталог** — весь текущий каталог точки текстом

Все изменения применяются сразу и следующий же `/start` их увидит.



1. Добавьте бота в чат/группу, куда должен приходить готовый заказ по конкретной точке (например, чат закупщика или общий чат снабжения).
2. Напишите в этот чат что угодно, затем узнайте `chat_id` через **@userinfobot** (перешлите ему сообщение из этого чата) или через `getUpdates` API.
3. Впишите `chat_id` в лист «Точки» файла `seed_products.xlsx`, столбец «Telegram Chat ID».
4. Перезапустите `import_products.py`.

Если `chat_id` не указан — бот присылает готовый заказ туда же, где его заполняли (удобно на старте/для тестов).

## Что уже умеет прототип

- Разные точки со своими категориями и списком позиций (как вы просили)
- Заказ по всем категориям сразу **или** по одной (например, только «Напитки»)
- Ввод «Факт» одним сообщением на всю категорию — числа по порядку, каждое на новой строке или через пробел; бот проверяет, что чисел ровно столько же, сколько позиций, и что все они ≥ 0 (при ошибке — просит отправить ещё раз)
- Автоматический расчёт заказа и статуса — 1-в-1 логика вашей таблицы
- Сводка перед отправкой с кнопками «Отправить» / «Заполнить заново» / «Отмена»
- Отправка готового заказа по шаблону конкретной точки в нужный чат
- История введённых остатков сохраняется в БД (`entries`) — на будущее пригодится для аналитики/отчётов по дням

## Что стоит добавить дальше (не реализовано в прототипе)

- **Точечная корректировка одной позиции** в сводке при заполнении заказа (сейчас «Заполнить заново» перезапускает всю категорию — самый простой и надёжный вариант для старта)
- Привязка сотрудника к своей точке (сейчас точку выбирают вручную каждый раз — как вы и описали)
- Более гибкие права доступа (сейчас `/admin` — это всё или ничего по списку `ADMIN_IDS`, без ролей)
- Напоминание в заданное время, если сотрудник забыл заполнить остатки
- Выгрузка истории (`entries`) обратно в Excel/отчёт

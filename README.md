# Бот заказов и тех.карт «по принципу таблицы остатков»

Telegram-бот с растущей архитектурой: сотрудник выбирает точку → попадает в
**меню точки** («📦 Заказ остатков» / «📖 Тех.карты», дальше будут добавляться
новые разделы) → в заказе выбирает категорию (или «все категории» — бот идёт
по всем подряд, объявляя переход к каждой следующей) → показывает позиции
**по одной**, с кнопками +1/+5/+10 и −1/−5/−10 (или «🔢 Ввести число» для
точного значения) → бот сам считает заказ и статус (🟢/🔴/🔵) — так же, как в
вашей Excel-таблице (`Заказ = Макс − Факт`) → показывает сводку с
возможностью точечно исправить любую позицию или заполнить заново →
отправляет готовый заказ с шаблоном конкретной точки.

Тех.карты — общий справочник блюд на все точки, с фото, ингредиентами и
способом приготовления; редактируются через `/admin`, видны любому сотруднику.

## Файлы

| Файл | Назначение |
|---|---|
| `seed_products.xlsx` | **Единственный файл, который редактируете руками.** Лист «Позиции» — точки/категории/наименования/мин/макс. Лист «Точки» — куда слать готовый заказ и заголовок шаблона. |
| `import_products.py` | Загружает `seed_products.xlsx` в базу `order_bot.db`. Запускать заново после каждого изменения Excel. |
| `db.py` | Работа с SQLite (точки, каталог, история введённых остатков). |
| `bot.py` | Сам бот (aiogram 3). |

## Запуск

```bash
pip install -r requirements.txt

# 1. Заполните seed_products.xlsx под свои точки/позиции
# 2. Загрузите каталог в базу:
python import_products.py seed_products.xlsx

# 3. Получите токен у @BotFather в Telegram и запустите:
export BOT_TOKEN="ваш_токен_от_BotFather"
export ADMIN_IDS="ваш_telegram_id"   # без этого /admin доступен всем — см. раздел ниже
python bot.py
```

Бот будет отвечать в личных сообщениях и в группах, куда его добавят.
Заказы — командой `/start`, управление каталогом (категории/позиции/мин-макс) — командой `/admin`.

## Управление каталогом: два способа

**1. Через Excel** (`seed_products.xlsx` → `import_products.py`) — удобно для массового
первоначального заполнения или переноса большого списка позиций.

**2. Через саму Telegram-команду `/admin`** — теперь можно на ходу, без Excel:
- создавать и удалять категории
- создавать и удалять позиции внутри категории
- менять мин./макс. остаток у любой позиции
- переименовывать категории и позиции

⚠️ Учтите: если после правок через `/admin` вы запустите `import_products.py`
заново, он **перезапишет** каталог точки содержимым Excel-файла — изменения,
сделанные через бота, будут потеряны для этой точки. Используйте один способ
как основной, второй — для точечных правок между обновлениями Excel.

### Кто может пользоваться `/admin`

Доступ ограничивается переменной окружения `ADMIN_IDS` (Telegram user id через
запятую, узнать свой id можно у **@userinfobot**):

```bash
export ADMIN_IDS="123456789,987654321"
```

Если `ADMIN_IDS` не задан — команда `/admin` доступна вообще всем, кто пишет
боту (удобно для тестов, но небезопасно в проде — обязательно задайте
переменную перед реальным запуском).

### Сценарий работы с `/admin`

1. `/admin` → выбрать точку (или создать новую)
2. В меню:
   - **➕ Категория** — ввести название, категория сразу появится в каталоге (даже пустая — позиции можно добавить позже)
   - **➕ Позиция** — выбрать категорию (или создать новую тут же) → название → единица измерения → мин и макс через пробел (например `2 10`)
   - **✏️ Мин/Макс позиции** — выбрать категорию → позицию → ввести новые мин/макс
   - **✏️ Переименовать позицию / категорию**
   - **🗑 Удалить точку / категорию / позицию** — с подтверждением (удаление точки или категории каскадно удаляет всё, что в ней, включая историю остатков)
   - **📋 Показать каталог** — весь текущий каталог точки текстом

Все изменения применяются сразу и следующий же `/start` их увидит.



1. Добавьте бота в чат/группу, куда должен приходить готовый заказ по конкретной точке (например, чат закупщика или общий чат снабжения).
2. Напишите в этот чат что угодно, затем узнайте `chat_id` через **@userinfobot** (перешлите ему сообщение из этого чата) или через `getUpdates` API.
3. Впишите `chat_id` в лист «Точки» файла `seed_products.xlsx`, столбец «Telegram Chat ID».
4. Перезапустите `import_products.py`.

Если `chat_id` не указан — бот присылает готовый заказ туда же, где его заполняли (удобно на старте/для тестов).

## Что уже умеет прототип

- Разные точки со своими категориями и списком позиций (как вы просили)
- Заказ по всем категориям сразу **или** по одной (например, только «Напитки»)
- Ввод «Факт» по одной позиции кнопками +1/+5/+10 / −1/−5/−10, с запасным вариантом «🔢 Ввести число» для точных/крупных значений
- При заказе «по всем категориям» бот объявляет переход к каждой следующей категории — удобно, если категории — это зоны склада (упаковка, кухня, касса и т.п.)
- Автоматический расчёт заказа и статуса — 1-в-1 логика вашей таблицы
- Сводка перед отправкой с кнопками «Отправить» / «Исправить одну позицию» / «Заполнить заново» / «Отмена»
- Отправка готового заказа по шаблону конкретной точки в нужный чат
- История введённых остатков сохраняется в БД (`entries`) — на будущее пригодится для аналитики/отчётов по дням

## Что стоит добавить дальше (не реализовано в прототипе)

- Привязка сотрудника к своей точке (сейчас точку выбирают вручную каждый раз — как вы и описали)
- Более гибкие права доступа (сейчас `/admin` — это всё или ничего по списку `ADMIN_IDS`, без ролей)
- Напоминание в заданное время, если сотрудник забыл заполнить остатки
- Выгрузка истории (`entries`) обратно в Excel/отчёт
- Переупорядочивание позиций внутри категории через `/admin` (сейчас порядок задаётся при создании; чтобы поменять — проще пересоздать через Excel-импорт)

## Понятность для сотрудника

- `/start` и `/help` показывают короткую инструкцию из 5 шагов + явное напоминание, что вводится именно «сколько есть сейчас», а не «сколько заказать» — это самое частое непонимание
- Экран каждой позиции показывает название, мин/макс и текущее указанное значение крупно — не нужно ничего печатать вручную в обычном случае
- Кнопка «🔢 Ввести число» — быстрый способ ввести точное или большое значение, не тыкая +1 много раз

## Навигация

- **Меню команд** — кнопка «Меню» рядом с полем ввода в Telegram всегда показывает `/start` и `/help` (у админов из `ADMIN_IDS` — ещё и `/admin`). Настраивается автоматически при запуске бота, вручную у @BotFather ничего задавать не нужно.
- **Постоянные кнопки внизу экрана** — «📦 Заполнить остатки» (и «⚙️ Админ-панель» у админов) закреплены под полем ввода, как в банковских ботах. Не нужно помнить команды — кнопка всегда на виду.
- **Обычные слова вместо команд** — бот понимает «остатки», «заказ», «привет», «меню», «начать» так же, как `/start`; «админ», «панель», «админка» так же, как `/admin` (для админов). Список слов — `START_TRIGGERS` и `ADMIN_TRIGGERS` в начале `bot.py`, можно дополнить своими.
- **Кнопки «⬅️ Категории» и «🏠 Точки»** есть на экране ввода остатков и на экране проверки перед отправкой — можно вернуться на шаг назад, не отменяя уже введённые данные и не начиная всё с нуля через /start.
- **«⬅️ Предыдущая позиция»** на экране ввода — можно вернуться и поправить только что введённое значение, не дожидаясь итоговой сводки.

Все эти способы (команда, кнопка, слово) работают как «аварийный выход» из любого места в диалоге — если сотрудник застрял на полпути заполнения, любой из них сразу вернёт в начало, не ломая уже сохранённые данные других точек.

## Точечное исправление ошибки

На экране проверки перед отправкой — кнопка **«🔧 Исправить одну позицию»**: показывает список позиций с текущими значениями, выбираете нужную, вводите одно новое число — сводка пересчитывается сразу. Пересобирать всю категорию заново из-за одной опечатки больше не нужно.

## Длинные категории и деление на зоны

Вместо одной большой категории «Хозы» из 21 позиции — заведите несколько категорий по смыслу («Упаковка», «Предкассовая зона», «Кухня» и т.п.), в `seed_products.xlsx` или через `/admin`. При заполнении «по всем категориям» бот сам объявит переход между ними («📍 Предкассовая зона»), и сотрудник будет точно понимать, на каком участке он сейчас — так же, как физически идёт по своим зонам, а не по алфавитному списку.

## Напоминания

Через `/admin` → точка → **«⏰ Напоминание»** можно задать время (ЧЧ:ММ по времени сервера), в которое бот проверит: заполнял ли кто-то остатки по этой точке сегодня. Если нет — пришлёт напоминание. «-» вместо времени выключает напоминание для точки.

- **Получатель по умолчанию** — тот же чат, куда уходит готовый заказ (`chat_id` точки). Отдельный получатель для напоминаний (`reminder_chat_id`) в базе уже предусмотрен, но настройка через `/admin` под него пока не выведена — если понадобится другой получатель, скажите, добавлю.
- Проверка идёт раз в минуту фоновым циклом внутри самого бота — отдельный сервис/cron не нужен.
- Если бот перезапускался ровно в момент отправки — напоминание за эту минуту может не уйти (лог отправленных напоминаний защищает только от дублей, не от пропуска при рестарте). Для ежедневного использования это не критично.
- Время сравнивается по локальному времени сервера, на котором крутится бот — если хостинг в другом часовом поясе, учтите это при вводе времени.


## ⚠️ Важно: постоянное хранилище на хостинге

`order_bot.db` создаётся во время работы бота, а не хранится в GitHub. **У bothost.ru есть готовое решение** — в их веб-терминале (иконка `>_` на карточке бота) написано прямым текстом:

> Данные (БД, файлы) храните в `/app/data` — эта папка сохраняется при перезапуске.
> Используйте: `DATA_DIR = os.getenv('DATA_DIR', '/app/data')`

`db.py` уже под это настроен: если задать переменную окружения `DATA_DIR`, файл базы будет создаваться внутри неё (`<DATA_DIR>/order_bot.db`) и переживёт пересборку.

**Что нужно сделать:**
1. В панели bothost.ru найдите переменные окружения вашего бота (там же, где заданы `BOT_TOKEN` и `ADMIN_IDS`)
2. Добавьте `DATA_DIR` со значением `/app/data`
3. Дождитесь пересборки — база будет создаваться уже в постоянной папке

Без `DATA_DIR` база продолжит создаваться заново при каждом деплое — на этот случай остаётся автовосстановление каталога из `seed_products.xlsx` (см. ниже), но правки через `/admin` и история остатков всё равно будут теряться.

**Что уже защищено в любом случае:** при старте бот сам проверяет — если каталог точек пуст, автоматически загружает его заново из `seed_products.xlsx` (та же логика, что в `import_products.py`). Значит даже без `DATA_DIR` каталог позиций восстановится сам, вручную ничего запускать не нужно.

**Что НЕ защищено этим авто-восстановлением (без `DATA_DIR`):**
- Точки/категории/позиции, добавленные вручную через `/admin` (их нет в `seed_products.xlsx`, значит после сброса базы они пропадут)
- История введённых остатков (`entries`) — а значит и подсказки «в прошлый раз было X», и логика напоминаний «уже заполняли сегодня»



## Быстрое массовое добавление точек/категорий/позиций

Для одной-двух правок — `/admin` удобен. Но если нужно добавить сразу много точек/категорий/позиций — быстрее сделать это в Excel, как в самом начале:

1. Откройте `seed_products.xlsx`, впишите все нужные точки (лист «Точки») и позиции (лист «Позиции») — можно копировать строки, это просто таблица
2. Загрузите обновлённый `seed_products.xlsx` на GitHub (замените файл), дождитесь пересборки бота
3. В боте: `/admin` → на экране выбора точки — кнопка **«📥 Импорт из seed_products.xlsx»** → подтвердите

Импорт применяется без захода на сервер: бот сам читает актуальный `seed_products.xlsx` (тот, что сейчас в репозитории) и загружает его в базу.

⚠️ **Важно:** импорт полностью заменяет каталог для каждой точки, которая указана на листе «Точки» файла — включая всё, что вы добавляли вручную через `/admin`. Точки, которых нет в файле, не затрагиваются. Используйте либо Excel, либо `/admin` как основной способ ведения каталога — смешивать оба способа для одной и той же точки стоит осторожно.

## Тех.карты (рецепты)

Общий справочник блюд, один на все точки — редактирует только админ, смотреть может любой сотрудник.

**Сотрудник:** команда `/recipes`, слово «рецепты»/«техкарты», либо кнопка «📖 Тех.карты» внизу экрана → выбирает блюдо → видит фото (если есть), ингредиенты с граммовкой и способ приготовления по шагам.

**Админ:** `/admin` → «📖 Тех.карты» → «➕ Добавить рецепт»:
1. Название блюда
2. Категория для навигации (например «Супы»), или «-» без категории
3. Ингредиенты — по одному на строку: `Название — количество ед` (например `Свёкла — 200 г`); нечисловые количества тоже работают: `Соль — по вкусу`
4. Способ приготовления — по шагам, каждый на новой строке; нумеровать не обязательно, бот сам пронумерует, если вы этого не сделали
5. Фото готового блюда — присылаете как обычное фото, или «-» чтобы пропустить

«📋 Список / удалить» показывает все рецепты, по каждому — карточка и кнопка удаления. Редактирование существующего рецепта пока не реализовано — для правки проще удалить и добавить заново.

## Меню точки (навигационный хаб)

После выбора точки сотрудник видит меню с разделами вместо того, чтобы сразу попадать в заказ:
- **📦 Заказ остатков** — ведёт в уже знакомый флоу (категория → ввод → сводка → отправка)
- **📖 Тех.карты** — общий справочник блюд (не зависит от точки, но доступен из меню каждой)
- **🏠 Сменить точку** — вернуться к выбору точки

Кнопка «⬅️ Меню точки» на экране выбора категории возвращает на этот хаб, не сбрасывая выбранную точку. Архитектура рассчитана на добавление новых разделов сюда же (база знаний, чек-листы, аттестация) — каждый новый модуль просто становится ещё одной кнопкой в этом меню.

## Разделы в тех.картах

При открытии «📖 Тех.карты» бот сначала показывает разделы меню (категории, которые вы задали при добавлении рецептов — например «Супы», «Выпечка»), а не сразу все блюда одним списком:
- **📖 Все блюда** — полный список, без фильтра
- Каждая категория — отдельной кнопкой
- **📄 Без категории** — появляется, только если есть рецепты без указанной категории
- Если вообще ни у одного рецепта нет категории, бот пропускает этот шаг и сразу показывает плоский список — лишнего экрана не будет, пока категории не понадобятся

## Категории тех.карт

При добавлении рецепта раздел меню теперь выбирается кнопками, а не вводится текстом — так исключаются опечатки, из-за которых «Супы» и «супы» стали бы разными разделами. Готовый список:

- Пельмени классические
- Пельмени заварные
- Вареники
- Салаты
- Супы
- Горячее
- Закуски

Если понадобится раздел, которого нет в списке — кнопка «➕ Другая категория» позволяет ввести любой текст; он запомнится и в следующий раз тоже появится кнопкой (список кнопок = встроенные разделы + все, что уже использовались). Изменить набор изначальных категорий можно в `bot.py`, константа `RECIPE_CATEGORY_PRESETS`.

## Обучение (обучение / хранение и маркировка / СанПиН — три раздела внутри)

Третий раздел меню точки — общий справочник на все точки, три фиксированных раздела:
- 📚 Обучение
- 🧊 Хранение и маркировка
- 🧼 СанПиН

**Сотрудник:** кнопка «📚 Обучение» внизу экрана, слово «база знаний»/«обучение»/«санпин»/«хранение»/«маркировка», либо через меню точки → «📚 Обучение» → выбирает раздел → выбирает статью → видит текст (и фото/схему, если есть).

**Админ:** `/admin` → «📚 Обучение» → выбирает раздел → «➕ Добавить статью»: заголовок → текст → фото (или «-» чтобы пропустить). «📋 Список / удалить» — просмотр и удаление статей внутри раздела.

Разделы фиксированы (три штуки, не расширяются кнопкой, в отличие от категорий тех.карт) — если понадобится больше разделов, это правится в `bot.py`, константа `KB_SECTIONS`.

## Массовая загрузка тех.карт и статей обучения

В `seed_products.xlsx` добавились ещё два листа — заполняете в Excel, загружаете тем же способом, что и позиции:

**Лист «Рецепты»:** Название | Категория | Ингредиенты | Способ приготовления
- Ингредиенты и шаги — каждая строка внутри своей ячейки на новой строке (в Excel: Alt+Enter)
- Формат ингредиента — как в `/admin`: `Свёкла — 200 г`, нечисловые тоже работают: `Соль — по вкусу`
- Категория — то же самое, что кнопки в `/admin` (`Пельмени классические`, `Супы` и т.п.) или своя

**Лист «Обучение»:** Раздел | Заголовок | Текст
- Раздел — один из трёх: `Обучение`, `Хранение и маркировка`, `СанПиН` (писать точно так же)

Загрузка — `/admin` → «📥 Импорт из seed_products.xlsx», та же кнопка, что и для позиций. В отличие от точек/позиций (которые **полностью заменяются** файлом), рецепты и статьи обучения **добавляются**: если рецепт с таким же названием и категорией (или статья с таким же заголовком в том же разделе) уже есть — строка просто пропускается, дубли не создаются. Можно смело гонять импорт повторно.

Фото к рецептам и статьям через Excel не грузятся — их можно добавить позже точечно через `/admin`, у каждого рецепта/статьи есть отдельная возможность прикрепить фото.

## Массовая загрузка тех.карт и статей обучения

Заполнять десятки блюд и статей по одной через переписку с ботом долго. Для объёма — как и с каталогом позиций — используйте Excel:

1. Откройте `seed_content.xlsx` — два листа: «Рецепты» и «Обучение»
2. **Рецепты:** Категория (одна из семи готовых, или своя) → Название → Ингредиенты (каждый на новой строке **внутри ячейки**, через Alt+Enter, в формате «Название — количество ед») → Способ приготовления (шаги, каждый на новой строке внутри ячейки)
3. **Обучение:** Раздел (строго «Обучение» / «Хранение и маркировка» / «СанПиН») → Заголовок → Текст статьи
4. Загрузите файл на GitHub, дождитесь пересборки
5. В боте: `/admin` → «📥 Импорт тех.карт и статей» → подтвердите

**Повторные запуски безопасны** — если название блюда или раздел+заголовок статьи уже существуют, содержимое обновится, а не задвоится. Проверено: второй запуск на том же файле не создаёт дублей.

⚠️ Фото через этот импорт не переносятся (Excel не хранит подходящие для Telegram файлы) — добавляются, как раньше, внутри бота через `/admin` → карточка блюда/статьи.
